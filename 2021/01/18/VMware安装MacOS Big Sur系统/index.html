VMware安装MacOS Big Sur系统

发表于 2021-01-18 分类于 各种折腾 Waline： 阅读次数： 本文字数： 2.2k 阅读时长 ≈ 2 分钟
经历了几次 MacOS 升级后带来的欲仙欲死的感觉后，只能用虚拟机先体验体验新版本了。

经历了几次 MacOS 升级后带来的欲仙欲死的感觉后，再也不敢轻易升级了，这次的 MacOS Big Sur 虽然很想体验一下，但是看着网上说的各种问题后，最终还是忍住了。毕竟电脑还要用于工作，把环境搞乱了恢复起来也还是一件很麻烦的事情。

但是就这么算了吗？显然不符合我们折腾的宗旨，一番研究之后准备祭出虚拟机大法，在家里的 Windowns 台式机上实验 VMware + MacOS。

准备工作

虚拟机 VMware 16

这里就不提供下载了，您可以自行搜索，至于版本个人倾向于用最新的，VMware Workstation 和 VMware Player 都可以。VMware Player 不仅是免费的，而且具有 VMware Workstatio 几乎相同的功能和特性。

Unlocker 工具

这个不是用来给 VMware Workstation 注册用的，而是一个可以让 VMware 能安装 MacOS 的插件，可以在 https://github.com/paolo-projects/unlocker/releases 上下载，目前最新的版本是 3.03；

MacOS 的镜像文件

可以去 [黑苹果动力：https://www.mfpud.com/macos/cdr/ 】看看，提供了各种版本的 CDR 镜像文件

如果你不是某度的会员，建议用天翼云，就算不是会员下载速度一样很快。

如果你想学习自制 MacOS 镜像文件，可以看看这篇文章《黑苹果 制作虚拟机CDR镜像（详细的教程，别再翻了！）》，文章地址：https://blog.csdn.net/qq_41855420/article/details/102750055

安装过程

VMware

VMware 的安装还是比较简单的。

Unlocker

在打补丁之前，VMware 并不支持安装 MacOS

在运行 Unlocker 之前，需要退出 VMware ，但实际上并不能完全退干净，所以还需要在任务管理器中，把带 vmware 开头的进程全部结束掉。

当然更安全的做法是在 服务 中将关于 VMware 的服务全部停止运行。

解压 unlocker.zip 文件然后右键点击 win-install.cmd 并使用管理员权限（必须是管理员）运行脚本，最好退出杀毒使用。

切记，win-install.cmd 必须使用管理员身份运行；

执行过程中会联网从 VMware 官方服务器下载多个依赖组件，时间取决于网速的情况，需要耐心等待。

脚本工作完成后，打开 VMware Workstation 新建虚拟机即可看到 Apple Mac OS 的选项。

MacOS

新建虚拟机，我用了 典型 模式

选择下载好的 Install macOS Big Sur 11.0.1 20B29 CDR.cdr 镜像文件，会提示 无法检测此光盘映像中的操作系统，不用理会，点击 下一步

选择 Apple Mac OS X，版本按我们下载的实际版本选择，然后点 下一步

直接用的默认名称就行

磁盘用的默认大小，存储改为了单个文件

到这就算安装完成了

运行设置

启动虚拟机

设置语言，我选了简体中文

选磁盘工具

选择 VMware Virtual SATA Hard Drive Media，然后点击抹掉

名称看个人喜好，格式用默认的 APFS

开始抹除

完成后退出 磁盘工具，选 重新安装macOS Big Sur

继续

同意

选择安装的磁盘，继续

安装过程挺花时间的

具体时间没统计

选 中国大陆

接下来就是一堆的初始化设置

辅助功能可以选 以后 来跳过

网络默认 本地网络

联网类型直接跳过了

选左下角的 以后

选 稍后设置

大功告成！

最后看下版本

解决问题

安装 VMware Tools 选项显示灰色

关闭虚拟机，打开虚拟机设置

CD/DVD -> 使用ISO映像文件 -> 浏览，如果 VMware 安装在 C盘，打开 C:\Program Files (x86)\VMware\VMware Workstation 目录，根据系统选择不同的 iso 文件

一般重启之后，桌面上就会有 VMware Tools

双击就可以安装了

之后就可以修改分辨率了，之前只能使用 1024*768

现在会随窗口变化

全屏可以到 1920*1080

如果你的电脑性能足够强，相当于又多了一台运行 MacOS 的苹果电脑，是不是很爽？
